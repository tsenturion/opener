# File Opener

Приложение для Windows, которое открывает PDF и изображения (`png`, `jpg`, `jpeg`, `bmp`, `gif`, `tif`, `tiff`, `webp`) во вкладках.

## Возможности

- Одновременная работа с несколькими файлами через верхние вкладки.
- Открытие отдельных файлов или всей папки через меню `Файл`.
- Сортировка файлов для листания через `Вид` -> `Сортировка`: по имени, дате изменения, типу или размеру.
- Для каждой вкладки: навигация `Файл назад` и `Файл вперед` по директории текущего файла.
- Для PDF: переход по страницам (`Предыдущая страница` / `Следующая страница`).
- Для каждого открытого файла отдельно: поворот, отражение и масштабирование.
- Верхняя панель управления для частых действий: открыть файлы или папку, листать файлы и страницы, повернуть на 90°.
- Редкие действия вынесены в меню `Преобразование`, масштабирование - в меню `Вид` и горячие клавиши.

## Управление масштабом

- `Ctrl` + колесо мыши: увеличить или уменьшить текущий файл.
- `Ctrl++`: увеличить.
- `Ctrl+-`: уменьшить.
- `Ctrl+0`: вернуть масштаб `100%`.

## Скачать exe

Готовый `FileOpener-*-windows-x64.exe` публикуется в GitHub Releases. Локально собирать `.exe` для публикации не нужно.

## Запуск из исходников

```powershell
python -m pip install -r requirements.txt
python main.py
```

## Сборка exe

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1
```

Результаты:

- `dist\FileOpener.exe`
- `FileOpener.exe` (копия в корне проекта)

## Release через GitHub Actions

Workflow `.github/workflows/release.yml` собирает Windows `.exe` на `windows-latest` и создает GitHub Release.

Вариант через тег:

```powershell
git tag v1.0.0
git push origin v1.0.0
```

Вариант вручную:

1. Откройте `Actions` в GitHub.
2. Запустите workflow `Release`.
3. Укажите тег, например `v1.0.0`.

Для публикации релиза workflow использует стандартный `GITHUB_TOKEN`, поэтому в настройках репозитория должны быть разрешены `Read and write permissions` для GitHub Actions.
